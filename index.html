Analisa Kompleksitas Algoritma

Studi Kasus: Mencari Jumlah Huruf Kapital (Iteratif vs Rekursif)

Metode Iteratif Looping

Jumlah Kapital: -
Waktu Eksekusi: - ms
Kompleksitas Waktu: O(n)
Kompleksitas Ruang: O(1)
function countIterative(str) { let count = 0; for (let i = 0; i < str.length; i++) { if (str[i] >= 'A' && str[i] <= 'Z') { count++; } } return count; }

Metode Rekursif Self-Calling

Jumlah Kapital: -
Waktu Eksekusi: - ms
Kompleksitas Waktu: O(n)
Kompleksitas Ruang: O(n)
function countRecursive(str, index = 0) { // Base Case if (index >= str.length) return 0; // Check current char const isCap = (str[index] >= 'A' && str[index] <= 'Z') ? 1 : 0; // Recursive Step return isCap + countRecursive(str, index + 1); }

Analisa Penggunaan Memori (Stack)

Iteratif (Efisien)

Hanya menggunakan satu variabel counter. Memori tetap stabil berapapun panjang kalimatnya.

Penggunaan Stack Memory
O(1)

Rekursif (Boros Memori)

Setiap karakter membuat "tumpukan" fungsi baru di memori sampai akhir kalimat tercapai.

Penggunaan Stack Memory
O(n)
Kesimpulan Analisa:
Meskipun kedua algoritma memeriksa setiap karakter sekali (Linear Time O(n)), Metode Iteratif jauh lebih baik untuk teks panjang.

Metode Rekursif memiliki kompleksitas ruang O(n). Jika Anda memasukkan teks yang sangat panjang (misal: > 10.000 karakter), metode rekursif akan menyebabkan Stack Overflow (Browser crash/error), sedangkan iteratif akan tetap berjalan lancar.
